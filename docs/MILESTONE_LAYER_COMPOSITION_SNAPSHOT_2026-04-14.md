# マイルストーン: Layer & Composition Snapshot System

> 2026-04-14 作成

## 目的

レイヤー単位、コンポジション単位の状態を「スナップショット」として保存・復元・分岐できるようにする。

Undo/Redo の履歴よりも強く、**「あの状態にもう一回戻したい」「この設定のバリエーションを試したい」** を実現する。

---

## 背景

制作中に以下のような場面が頻発する：

- 「この演出パターンどうだったっけ？」
- 「色調全部変えた版も試したい」
- 「事故る前に戻したい」
- 「このエフェクト設定よかったから一旦保存」

現在の Undo/Redo は**線形**で、分岐して保存できない。
プロジェクトを別名保存して複製する手法はあるが、**重すぎる**。

---

## 概念

### Composition Snapshot（メイン機能）

**何を保存するか:**
- レイヤー構造（追加/削除/順序）
- 各レイヤーのパラメータ全量
- キーフレーム全量
- エフェクトスタック
- タイミング情報
- マット関係・親子関係

**= コンポジション丸ごとの状態**

**使いどころ:**
- 「この演出パターンどう？」
- 「色調全部変えた版も試したい」
- 「事故る前に戻したい」

**特徴:**
- AEの「履歴」より強い（分岐できる）
- DaVinciの「タイムライン複製」に近い
- Gitブランチ的

---

### Layer Snapshot（サブ機能）

**何を保存するか:**
- 単一レイヤーの
  - Transform（位置/スケール/回転/不透明度）
  - Effectパラメータ
  - キーフレーム

※ 構造変更は含まない

**使いどころ:**
- 「この色調よかったから一旦保存」
- 「このエフェクト設定A/B試したい」
- 「微調整戻したい」

**特徴:**
- 軽い
- UIに近い（プリセット的）
- Undoより長期保存できる

## フェーズ設計

### Phase 1: Snapshot Data Contract

**目的:**
スナップショットのデータ構造を定義する。

**作業項目:**
- `CompositionSnapshot` struct
  - レイヤーリスト
  - レイヤー間関係（親子/matte）
  - コンポジション設定
- `LayerSnapshot` struct
  - Transform全量
  - Effectパラメータスナップ
  - キーフレーム一覧
- `SnapshotMetadata`
  - タイトル
  - タイムスタンプ
  - タグ/メモ
  - チャンク（差分/完全）

**完了条件:**
- シリアライズ/デシリアライズ可能
- JSON バイナリ両対応の基礎がある

---

### Phase 2: Capture & Restore

**目的:**
スナップショットの作成と復元を実装する。

**作業項目:**
- `CompositionSnapshotter::Capture()`
  - 現在のコンポ状態をキャプチャ
  - 全レイヤーをトラバース
  - 関係を記録
- `CompositionSnapshotter::Restore()`
  - スナップショットから状態を復元
  - 既存状態を置き換え or 一時プレビュー
- `LayerSnapshotter::Capture()`
  - 単一レイヤーの現在状態をキャプチャ
- `LayerSnapshotter::Restore()`
  - レイヤーにスナップショットを適用
  - パラメータ上書き

**完了条件:**
- キャプチャ→復元で同一状態に戻れる
- 複数スナップショットを保存可能

---

### Phase 3: Snapshot Manager

**目的:**
スナップショットの保存・一覧・削除・検索を管理する。

**作業項目:**
- `SnapshotManager` クラス
  - スナップショット一覧取得
  - 保存/削除
  - タグ/メモ付与
  - 時系列ソート
  - 検索フィルタ
- ストレージ設計
  - プロジェクトファイル内保存
  - 外部ファイルエクスポート（.snap）
  - 容量制限/エビクション

**完了条件:**
- UIからスナップショット操作可能
- プロジェクト保存/読み込みでスナップショットも永続化

---

### Phase 4: Branch & Compare

**目的:**
スナップショットの分岐・比較機能を実装する。

**作業項目:**
- ブランチ的運用
  - スナップショットから分岐して別状態として編集
  - スナップショット名の衝突回避
- 比較機能
  - 2つのスナップショットの差分表示
  - レイヤー単位差分ハイライト
  - プレビュー切り替え
- マージ
  - スナップショット間のパラメータマージ（限定的）

**完了条件:**
- スナップショットA/B比較ができる
- 分岐して別バージョンを試せる

---

## Non-Goals

- プロジェクト全体のバージョン管理（Git統合）
- 完全なマージコンフリクト解決
- リアルタイム共同編集
- 無限履歴保持（容量制限はPhase 3で扱う）

---

## 技術方針

### シリアライズ

- JSON（可読性重視、デバッグ用）
- バイナリ（容量/速度重視、本番用）
- 両方の相互変換可能

### ストレージ

```text
Project
├─ Composition
│  ├─ Snapshots/
│  │  ├─ snap_001.snap
│  │  ├─ snap_002.snap
│  │  └─ ...
│  └─ Layers/
│     └─ layer_id_123/
│        ├─ snap_001.snap
│        └─ ...
```

### 差分 vs 完全

- 初回: 完全スナップショット
- 2回目以降: 差分のみ保存（容量節約）
- 復元時は初回+差分を適用
